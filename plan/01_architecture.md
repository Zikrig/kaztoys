# Общая архитектура бота

## 1.1 Точки входа

- **Реферальная ссылка** — вход по рефералу
- **Ссылка Instagram / реклама** — вход не по рефералу
- **Поиск в Telegram** — просто найти бота и войти
- **Команда** `/start`

## 1.2 Первый запуск (onboarding)

1. Бот отправляет **дисклеймер** (нет ответственности за товар, незаконные действия и т.д.).
2. Бот отправляет **видео-инструкцию** (mp4) по использованию.
3. Сообщение: «Готовы ли вы начать поиск игрушек?»
4. Кнопка **«Начать поиск»** → переход в флоу поиска (см. [03_search.md](03_search.md)).

## 1.3 Главное меню (дефолтный режим)

- Пока пользователь в главном меню — ему **автоматически приходят**:
  - новые заявки других пользователей (если включена подписка/доступ);
  - уведомления, если его отклик выбрали по какой-то заявке.
- При возврате в меню: если есть **неподтверждённые мэтчи** — отдельное сообщение: «У вас есть неподтверждённые мэтчи. Пожалуйста сделайте с ними что нибудь».
- **Таймаут неактивности:** через 10 минут без действий — возврат в главное меню из любого состояния (создание заявок, вводы и т.д.).

**Кнопки в зоне ввода текста (главное меню):**

- Выставить свою игрушку
- Начать поиск
- Мои заявки
- Мои отклики
- Открыть неподтверждённые сделки
- Моя подписка
- Связаться с поддержкой

Кнопка «Начать поиск» ведёт в флоу поиска: опрос (категория, возраст, район) → показ заявок по одной. Остальное меню без изменений.

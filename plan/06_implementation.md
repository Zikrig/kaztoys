# План реализации и структура проекта

## 6.1 Этапы реализации

### Этап 1 — Ядро и онбординг

- [ ] Проект Python (venv, зависимости: aiogram / ptb, aiohttp, БД).
- [ ] Подключение к Telegram Bot API, обработка `/start`, реферальные метки.
- [ ] Дисклеймер, видео-инструкция, «Готовы начать?» → «Начать поиск».
- [ ] Главное меню: кнопки, хранение состояния (FSM), таймаут 10 мин в главное меню.

### Этап 2 — Заявки

- [ ] Генерация уникального кода, приём фото с кодом.
- [ ] Шаги: Категория (5 кнопок) → Возраст (5 кнопок) → Описание → Подтверждение.
- [ ] Сохранение заявки в БД (код, фото, категория, возраст, район, описание, user_id, дата).
- [ ] Редактирование заявки (заново с фото по тому же коду).
- [ ] Раздел «Мои заявки»: список, «Посмотреть отклики», «Закрыть заявку».

### Этап 3 — Поиск

- [ ] Сохранение параметров поиска пользователя (категория, возраст, район).
- [ ] Опрос при первом «Начать поиск» (3 шага); при повторном — сразу показ.
- [ ] Показ одной заявки (фото, имя, сделки, код, категория, возраст, район, описание).
- [ ] Кнопки: Предложить обмен | Дальше | В главное меню; «Изменить параметры поиска».
- [ ] Логика «заявки кончились» + сообщение и кнопки.
- [ ] Подсказка после 7 «Дальше» (один раз за сессию).

### Этап 4 — Отклики и матчи

- [ ] Отклик: выбрать имеющуюся игрушку / создать новую; код + фото + текст; подтверждение.
- [ ] Уведомления: «На вашу заявку поступил отклик», «Ваш отклик выбрали» + контакты.
- [ ] Кнопки «Выбрать» у откликов, «Откликнуться» у входящих откликов.
- [ ] Формат «ПОМНИТЕ???» и подтверждение сделки / «Сделка не состоялась».
- [ ] Счётчик подтверждённых сделок.

### Этап 5 — Рассылки и подписка

- [ ] В главном меню: рассылка новых заявок (по подписке/доступу) в формате одной заявки.
- [ ] Платная подписка (доступ на 2 недели): оплата, хранение срока, ограничение без подписки.
- [ ] «Моя подписка» и «Связаться с поддержкой».

### Этап 6 — Доработки и уведомления

- [ ] Уведомления о новых подходящих заявках (когда пользователь долистал до конца и сохранены фильтры).
- [ ] Напоминание через 24 часа о неподтверждённом мэтче (если нужно).
- [ ] Удаление заявки: уведомление пользователю, который уже выбрал эту заявку.

### Этап 7 — Тестирование и деплой

- [ ] Ручное тестирование всех сценариев.
- [ ] Список районов Астаны внесён в бота.
- [ ] Деплой (сервер/VPS, systemd/docker), логи, мониторинг.

## 6.2 Структура проекта (рекомендуемая)

```
kaztoys/
├── .env                    # BOT_TOKEN, DATABASE_URL, REFERRAL_PARAM...
├── requirements.txt
├── README.md
├── plan/                   # План разработки (эти файлы)
├── docs/                   # Дизайн-документы (БД, логика, истории)
├── bot/
│   ├── __init__.py
│   ├── main.py             # точка входа, polling/webhook
│   ├── config.py           # настройки из env
│   ├── handlers/
│   │   ├── __init__.py
│   │   ├── start.py        # /start, дисклеймер, видео, меню
│   │   ├── menu.py         # главное меню, таймаут
│   │   ├── listing.py      # создание/редактирование заявки
│   │   ├── search.py       # опрос, показ по одной, Дальше, Изменить параметры
│   │   ├── response.py     # отклики, выбор своей игрушки, отправка
│   │   ├── matches.py      # отклики по заявке, выбор, мэтч, подтверждение сделки
│   │   ├── subscription.py
│   │   └── support.py
│   ├── keyboards/
│   │   ├── categories.py   # категории и возраст
│   │   ├── districts.py    # районы Астаны
│   │   └── menu.py
│   ├── services/
│   │   ├── listing.py      # CRUD заявок, коды
│   │   ├── search.py      # фильтрация, пагинация по одной
│   │   ├── response.py
│   │   └── subscription.py
│   ├── models/
│   │   ├── user.py
│   │   ├── listing.py
│   │   ├── response.py
│   │   └── deal.py
│   └── middlewares/
│       ├── auth.py
│       └── inactivity.py   # возврат в меню через 10 мин
├── migrations/             # Alembic
└── tests/
```

При необходимости этапы можно разбить на спринты и назначить сроки.
